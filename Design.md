# Test Buddy AI：程序员的智能测试助手
Test Buddy AI 是一个面向程序员用户的创新项目，旨在通过AI技术自动化代码测试流程。用户只需使用Docker在本地运行该项目，并接入自己的代码仓库，Test Buddy AI便会持续主动地为项目添加测试代码、运行测试并报告问题，帮助用户提升代码质量和测试覆盖率。

## 项目简介
Test Buddy AI 是一个智能测试助手，专为程序员设计。它通过分析代码仓库，自动生成测试用例，运行测试，并提供详细的问题报告和修复建议。项目利用先进的AI技术（如大型语言模型LLM和检索增强生成RAG），结合高效的后端服务（FastAPI）和简洁的前端界面（React），为用户提供无缝的测试体验。借助Docker，用户可以轻松在本地部署和运行该项目。

## 核心功能
* 自动测试生成Test Buddy AI分析代码结构，自动生成单元测试和集成测试用例，减少手动编写测试的工作量。
* 持续测试运行项目会定期运行测试用例，监控代码变更，并在每次更新后自动调整测试内容。
* 问题检测与报告识别测试失败、潜在bug或低测试覆盖率的区域，并提供具体的修复建议。
* 本地运行支持通过Docker，用户可以在自己的机器上运行Test Buddy AI，无需依赖云服务，保护代码隐私。

## 使用方式
1. 安装与运行用户通过Docker拉取项目镜像，并在本地启动容器。安装步骤简单，只需几条命令即可完成。
2. 接入代码仓库将本地代码仓库的路径配置到Test Buddy AI中，项目会自动扫描并开始工作。
3. 查看结果通过内置的React界面，用户可以实时查看测试覆盖率、测试结果和问题报告。

## 技术实现
### 技术栈
* 前端：React提供直观的仪表板和配置页面。
* 后端：FastAPI高效处理测试生成、运行和报告。
* AI核心：  
    * LLM：分析代码并生成测试用例。  
    * RAG：从文档中检索信息，支持测试生成。
* 容器化：Docker确保本地部署的便捷性和一致性。  
* 测试工具：pytest（支持多种语言的测试框架）执行生成的测试用例。
### 工作流程
1. 用户启动Docker容器并配置代码仓库路径。  
2. Test Buddy AI使用LLM分析代码，识别需要测试的部分。  
3. 结合RAG技术从文档中提取知识，生成测试用例。  
4. 自动运行测试，记录结果并生成报告。  
5. 持续监控代码变更，更新测试并重复流程。

### 项目优势
* 自动化：减少手动编写测试的负担，提高开发效率。  
* 智能化：利用AI提供精准的测试用例和问题建议。  
* 本地化：Docker支持本地运行，保护代码安全。  
* 易用性：简洁的界面和清晰的报告，适合所有程序员使用。
